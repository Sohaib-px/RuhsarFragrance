# Email Setup Guide for Ruhsar Contact Form

## Step 1: Install Nodemailer
```bash
npm install nodemailer @types/nodemailer
```

## Step 2: Set Up Gmail App Password

1. Go to your Google Account: https://myaccount.google.com
2. Enable **2-Factor Authentication** if not already enabled
3. Go to https://myaccount.google.com/apppasswords
4. Select "Mail" and "Windows Computer"
5. Google will generate a 16-character app password
6. Copy this password

## Step 3: Create .env.local File

Create a `.env.local` file in the root of your project (same level as `package.json`):

```
EMAIL_USER=ruhsarfragrancepk@gmail.com
EMAIL_PASSWORD=xxxx xxxx xxxx xxxx
```

**Note**: Use the 16-character app password generated by Google (spaces don't matter when pasting)

## Step 4: Test the Form

1. Run your development server: `npm run dev`
2. Navigate to the Contact section
3. Fill in and submit the form
4. Check if emails arrive at ruhsarfragrancepk@gmail.com

## Troubleshooting

### "SMTP Authentication failed"
- Verify EMAIL_USER and EMAIL_PASSWORD are correct
- Check that the app password is not expired
- Ensure 2FA is enabled on your Google account

### "EMAIL_USER is undefined"
- Make sure `.env.local` file exists in the project root
- Restart the development server after adding/modifying `.env.local`

### Emails not arriving
- Check spam/junk folder
- Verify the recipient email address is correct
- Check server logs for errors

## Security Notes

- Never commit `.env.local` to version control
- `.env.local` is already in `.gitignore`
- Use app passwords instead of your actual Gmail password
- For production, use a proper email service like SendGrid, Mailgun, or AWS SES
